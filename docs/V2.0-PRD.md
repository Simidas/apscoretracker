## 0. 结论先行

- **做什么**: 从纯前端 localStorage 工具升级为**有用户系统、云端数据存储、订阅付费**的完整产品
- **为什么做**: V1.x 验证了"追踪"需求存在，但 localStorage 导致数据易丢失、无法跨设备、无法商业化
- **一句话价值**: 学生登录后，任何设备上的模考记录都能同步，升级 Pro 解锁无限追踪和高级分析
- **明确不做**: 教师班级版、AI 作文评分、社交排行榜、移动端 Native App、多语言（仍英文-only）

## 1. 背景与动机

### V1.x 现状

| 指标 | 状态 |
|------|------|
| 技术栈 | Next.js 15.5 + React 18 + Cloudflare Workers |
| 存储 | localStorage（浏览器本地） |
| 用户系统 | ❌ 无 |
| 付费 | ❌ 无 |
| 科目 | 10 门 |
| 核心功能 | 分数计算、历史记录、进步曲线、热力图、目标分数、学习建议、打印样式 |

### V1.x 核心痛点

1. **数据易丢失**: 清缓存、换浏览器、换设备 = 数据全没
2. **无法商业化**: 没有用户身份，无法区分 Free/Pro，无法收费
3. **使用门槛隐形**: 用户以为"打开即用"，实则数据绑死当前浏览器
4. **无留存抓手**: 无法触达用户（无邮箱）、无法分析用户行为（无账号）

### V2.0 目标

- 用户登录后数据自动云端同步
- Free/Pro 分层，Pro 订阅解锁高级功能
- 为后续迭代（社交分享、AI 建议、家长报告）打下账号基础

### 2.2 Clerk 配置

- **登录方式**: Google OAuth（首选）、Email + 验证码（备选）
- **用户字段**: 除 Clerk 默认外，扩展 `subscription_tier`（free/pro）
- **Session**: JWT，有效期 7 天，自动刷新
- **成本**: 前 10K MAU 免费，超出 $0.02/MAU

### 2.3 D1 Schema

```sql
-- 用户表（由 Clerk 管理，D1 只存扩展字段）
CREATE TABLE users (
  id TEXT PRIMARY KEY,           -- Clerk user_id
  email TEXT UNIQUE NOT NULL,
  subscription_tier TEXT DEFAULT 'free',  -- 'free' | 'pro'
  stripe_customer_id TEXT,
  stripe_subscription_id TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 考试记录表（替代 localStorage）
CREATE TABLE exam_records (
  id TEXT PRIMARY KEY,           -- uuid v4
  user_id TEXT NOT NULL,
  subject TEXT NOT NULL,         -- 'ap-lang' | 'ap-psych' | ...
  date TEXT NOT NULL,            -- ISO 8601
  mcq_score REAL NOT NULL,
  frq_score REAL NOT NULL,
  total_score REAL NOT NULL,
  ap_score INTEGER NOT NULL,     -- 1-5
  breakdown TEXT,                -- JSON: {"topic": score}
  notes TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 目标分数表（替代 localStorage apst_targets）
CREATE TABLE target_scores (
  user_id TEXT NOT NULL,
  subject TEXT NOT NULL,
  target_score INTEGER NOT NULL DEFAULT 3,  -- 1-5
  PRIMARY KEY (user_id, subject),
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 索引
CREATE INDEX idx_exam_records_user_subject ON exam_records(user_id, subject);
CREATE INDEX idx_exam_records_date ON exam_records(date);
```

### 2.4 Stripe 产品配置

| 字段 | Pro Monthly | Pro Yearly |
|------|-------------|------------|
| product_key | apst_pro_monthly | apst_pro_yearly |
| price | $4.99/月 | $39.99/年 |
| 权益 | 无限科目、无限记录、跨设备同步、CSV导出、无广告、高级打印报告 | 同上 |

**Webhook 事件处理:**
- `checkout.session.completed` → 创建/更新用户订阅记录
- `invoice.paid` → 续期确认
- `invoice.payment_failed` → 标记待取消，宽限期 3 天
- `customer.subscription.deleted` → 降级为 free（保留数据，禁止新增）

## 3. 功能规划

### 3.1 P0 — 用户系统（登录/注册/会话）

**需求描述**: 用户可通过 Google 一键登录或邮箱验证码登录，登录后数据自动同步到云端。

**交互设计:**
- 页面右上角显示 "Sign In" 按钮（未登录）或用户头像（已登录）
- 点击 Sign In → Clerk 弹出登录框 → 选择 Google / Email
- 首次登录 = 自动注册，创建 D1 users 记录（tier='free'）
- 追踪器核心功能（保存记录、查看历史、进步曲线）**强制要求登录**，未登录用户仅可浏览 Landing Page 和试用计算器

**验收标准:**
- [ ] Google OAuth 一键登录成功
- [ ] Email 验证码登录成功
- [ ] 登录后页面刷新，用户状态持久
- [ ] 登出后清除本地 session，回到匿名状态
- [ ] 未登录用户可使用 Landing Page 和试用计算器，但无法保存数据

---

### 3.2 P0 — 云端数据存储（D1）

**需求描述**: 登录用户的所有 exam_records、target_scores 读写从 localStorage 切换到 D1。

**数据流:**
```
已登录用户:
  保存记录 → API Route (/api/records) → D1 insert → 返回确认 → 前端更新
  读取记录 → API Route (/api/records?subject=ap-lang) → D1 select → 返回列表

未登录用户:
  仅可浏览 Landing Page 和使用试用计算器
  保存/查看历史等功能需登录后使用
```

**API Routes:**
| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/records` | GET | 获取当前用户所有记录（可选 ?subject=xxx 过滤） |
| `/api/records` | POST | 创建新记录 |
| `/api/records/[id]` | DELETE | 删除单条记录 |
| `/api/targets` | GET | 获取目标分数 |
| `/api/targets` | POST | 设置/更新目标分数 |
| `/api/user` | GET | 获取当前用户订阅状态 |

**验收标准:**
- [ ] 登录用户保存记录后，刷新页面数据仍在
- [ ] 换浏览器登录同一账号，数据自动同步
- [ ] 未登录用户不受影响，仍用 localStorage
- [ ] API 响应时间 < 500ms（边缘部署）

### 3.3 P0 — Free / Pro 分层限制

**验收标准:**

### 3.4 P0 — Stripe 订阅支付（内嵌 Elements）

**需求描述**: 用户可通过 Stripe Elements 内嵌表单购买 Pro 订阅，支持月付/年付，无需跳转外部页面。

**交互流程:**
```
用户点击 "Upgrade to Pro"
  → 弹出 Upsell Modal，内嵌 Stripe Elements 支付表单
  → 用户填写卡号 → 前端调用 stripe.confirmPayment()
  → 支付成功 → 前端调用 /api/stripe/confirm 确认
  → 后端更新 users.subscription_tier = 'pro'
  → Modal 关闭，显示 "Welcome to Pro!"
```

**Stripe Elements 配置:**
- 使用 `@stripe/stripe-js` + `@stripe/react-stripe-js`
- Payment Element 支持卡号、Apple Pay、Google Pay
- 客户端创建 PaymentIntent → 确认支付 → 服务端 webhook 处理

**管理功能:**
- 用户头像下拉菜单新增 "Manage Subscription" → 跳转 Stripe Customer Portal
- 支持用户自助取消、更新支付方式、查看账单

**验收标准:**
- [ ] Stripe Elements 内嵌支付表单正常渲染
- [ ] 卡号输入、3D Secure、支付成功完整流程
- [ ] 支付成功后用户状态实时变为 Pro
- [ ] Webhook 处理订阅生命周期（创建/续期/取消/过期）
- [ ] 订阅取消后用户到期降级为 Free
- [ ] Customer Portal 可正常管理订阅

## 5. 安全与合规

### 5.1 安全

- **API 认证**: 所有 `/api/*` 路由（除 webhook）校验 Clerk session，未登录返回 401
- **Webhook 安全**: Stripe webhook 验证 `stripe-signature`，拒绝伪造请求
- **Rate Limit**: Workers KV 计数，API 路由 100 次/分钟/IP
- **CORS**: 仅允许同域请求（Cloudflare Workers 默认）
- **SQL 注入**: D1 使用参数化查询（Wrangler D1 API 天然支持）

### 5.2 合规更新

| 文档 | 更新内容 |
|------|----------|
| 隐私政策 | 新增：Clerk 处理登录数据、D1 存储考试记录、Stripe 处理支付、数据保留期限 |
| 服务条款 | 新增：订阅条款、退款政策、降级规则、账户删除 |
| Cookie 政策 | 新增：Clerk session cookie 说明 |

## 7. 风险与回滚

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| Clerk 10K MAU 免费档超限 | 🟡 中 | 监控 MAU，接近时评估自切 Auth.js |
| D1 免费档 5GB 超限 | 🟡 中 | 单条记录 < 1KB，5GB ≈ 500万条，早期不可能 |
| Stripe webhook 丢失 | 🟡 中 | 配置 Stripe Dashboard 备用通知，人工兜底 |
| 强制登录导致新用户流失 | 🔴 高 | Landing Page 保留试用计算器，降低体验门槛；监控注册转化率 |
| 未登录用户功能退化引发投诉 | 🟢 低 | V2.0 全新上线，无存量用户；未登录用户明确引导注册 |
| 支付合规（税务/VAT） | 🟡 中 | Stripe Tax 自动处理，或用户自报；马来法确认 |

**回滚方案:**
- 代码层: git revert V2.0 commit，回退到 V1.1
- 数据层: D1 数据保留，如需降级可导出用户数据
- 支付层: Stripe 订阅可批量退款/取消
